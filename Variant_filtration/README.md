# Variant Filtration Pipeline

A Nextflow DSL2 pipeline for post-processing VCF files generated by germline variant callers. It applies GATK4 hard filtering following [GATK Best Practices](https://gatk.broadinstitute.org/hc/en-us/articles/360035531112), producing clean, indexed VCF files ready for downstream annotation or clinical interpretation.

This pipeline is designed to run after [variant_caller.nf](../variant_caller.nf) or any pipeline that produces raw GATK HaplotypeCaller VCF output.

---

## Pipeline overview

```
Raw VCF (HaplotypeCaller output)
        │
        ▼
SelectVariants         ── Split into SNPs and indels separately
        │
        ▼
VariantFiltration      ── Apply GATK hard filters per variant type
        │                  SNPs:   QD, QUAL, SOR, FS, MQ, MQRankSum, ReadPosRankSum
        │                  Indels: QD, QUAL, FS, ReadPosRankSum
        ▼
MergeVcfs              ── Merge filtered SNPs and indels into one VCF
        │                  (sorting handled internally by MergeVcfs)
        ▼
IndexFeatureFile       ── Index final curated VCF
        │
        ▼
Curated VCF + index
```

---

## Requirements

- [Nextflow](https://www.nextflow.io/) >= 23.0.0
- Docker or Singularity
- GATK 4.x container

---

## Usage

**Run locally with Docker:**
```bash
nextflow run variant_filtration.nf \
    -profile docker \
    --vcf_files 'results/raw_variant_calling/*.{vcf,vcf.idx}' \
    --reference /path/to/reference.fa \
    --outdir results
```

**Run on HPC with SLURM + Singularity:**
```bash
nextflow run variant_filtration.nf \
    -profile slurm,singularity \
    --vcf_files 'results/raw_variant_calling/*.{vcf,vcf.idx}' \
    --reference /path/to/reference.fa \
    --outdir results
```

**WES or panel — restrict to target regions:**
```bash
nextflow run variant_filtration.nf \
    -profile docker \
    --vcf_files 'results/raw_variant_calling/*.{vcf,vcf.idx}' \
    --reference /path/to/reference.fa \
    --region_intervals /path/to/targets.bed \
    --outdir results
```

**Resume a failed run:**
```bash
nextflow run variant_filtration.nf -resume [OPTIONS]
```

## Parameters

| Parameter | Description | Default |
|-----------|-------------|---------|
| `--vcf_files` | Glob pattern for input VCF + index pairs | `results/raw_variant_calling/*.{vcf,vcf.idx}` |
| `--reference` | Path to reference FASTA | required |
| `--region_intervals` | BED file for target regions (WES/panels) | off |
| `--outdir` | Output directory | `results` |

## Hard filters applied

### SNPs

| Filter name | Expression | Rationale |
|-------------|------------|-----------|
| QD2 | QD < 2.0 | Low quality by depth — likely false positive |
| QUAL30 | QUAL < 30.0 | Low overall variant quality |
| SOR3 | SOR > 3.0 | Strand odds ratio — strand bias |
| FS60 | FS > 60.0 | Fisher strand — strand bias |
| MQ40 | MQ < 40.0 | Low mapping quality of reads |
| MQRankSum-12.5 | MQRankSum < -12.5 | Mapping quality rank sum test |
| ReadPosRankSum-8 | ReadPosRankSum < -8.0 | Read position rank sum test |

### Indels

| Filter name | Expression | Rationale |
|-------------|------------|-----------|
| QD2 | QD < 2.0 | Low quality by depth |
| QUAL30 | QUAL < 30.0 | Low overall variant quality |
| FS200 | FS > 200.0 | Strand bias — threshold higher for indels |
| ReadPosRankSum-20 | ReadPosRankSum < -20.0 | Read position rank sum test |

Filtered variants are **not removed** — they are flagged in the FILTER column of the VCF. Passing variants carry `PASS`.

## Output

```
results/
├── intermediate/
│   ├── selected/          # Per-sample SNP and indel VCFs (pre-filter)
│   └── filtered/          # Per-sample filtered SNP and indel VCFs
└── curated/               # Final merged and indexed VCFs
    ├── *.curated.vcf      # One file per sample — ready for annotation
    └── *.curated.vcf.idx  # GATK index
```

---

## Notes

- Input VCFs must be paired with their index file (`.vcf` + `.vcf.idx`). If the index is missing, generate it first with `gatk IndexFeatureFile --input sample.vcf`.
- `SortVcf` is not needed before `MergeVcfs` — GATK `MergeVcfs` handles sorting internally.
- These are **hard filters**, not VQSR. Hard filtering is recommended when the number of samples is too small to train the VQSR model (fewer than 30 WGS samples or fewer than 100 WES samples).
- The curated VCFs are ready for annotation with VEP, snpEff, or as input to the Sarek `--step annotate` mode.
